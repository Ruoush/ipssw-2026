0 – 10 min

Welcome & Framing: What Is "Just in Time Fidelity"?

10 min

The Core Idea

High-fidelity simulation tools — mannequins, moulage, task trainers — are expensive and often require lead time to prepare. Just in time fidelity is the practice of creating purpose-built, scenario-specific tools quickly and cheaply, exactly when they're needed.

AI chatbots can now generate working HTML web apps in seconds. A browser is all learners need to run them. This unlocks a new category of simulation support: lightweight, interactive, instantly deployable digital tools.

💡
Facilitator talking point: "You've been building just-in-time solutions your whole career — a quick moulage, a printed checklist, a prop from the supply closet. This is the same instinct, supercharged. You describe what you need, and the AI builds it."

What Can HTML Simulation Tools Do?

  • Step-by-step procedural coaching (like today's AED app)
  • Interactive checklists and decision trees
  • Timer-based scenarios with alerts
  • Patient assessment screens with vital signs
  • Medication dosage calculators
  • Visual anatomy or equipment diagrams with clickable labels
  • Team role assignment wheels

What HTML Is (In Plain Language)

HTML is the language all websites are built with. An HTML file is just a text file that your browser knows how to display. You don't need a server, an app store, or any installation. You open the file — it works.

You will not be writing HTML today. You will be describing what you want to an AI, and the AI will write it for you.

20 – 35 min

How to Talk to AI: The 5 Prompt Templates

15 min

You don't need to know how to code to get good code from an AI. You need to know how to describe what you want clearly. These 5 templates cover almost every situation you'll encounter.

📋
Works with any AI chatbot — Claude, ChatGPT, Gemini, or others. Open one now and follow along.
1
The Start From Scratch Prompt Use when: starting a new tool
Create a single-file HTML simulation tool for medical training. What it does: [describe the scenario in plain language] Who will use it: [learner type, e.g. nursing students, paramedics] Key features I need: - [feature 1] - [feature 2] - [feature 3] Keep it simple and easy to use. No external libraries. Make it work when opened directly in a browser.
Tip: More detail = better results. Describe the scenario like you're briefing a colleague who has never seen it before.
2
The Make It Better Prompt Use when: the output is close but not quite right
This is good, but I'd like to make a few changes: - [specific thing to change] - [specific thing to add] - [specific thing to remove] Please update the code and give me the full file again.
Tip: Always ask for "the full file again" — this gives you a complete, working file to copy/paste and save.
3
The Add a Feature Prompt Use when: you want one specific new capability
Add [specific feature] to this HTML file. [Paste your current HTML code here] The feature should: [describe exactly what it should do]. Please give me the complete updated file.
Tip: One feature at a time works better than listing many at once. You can always add more after.
4
The Fix the Bug Prompt Use when: something isn't working
Something isn't working in my HTML file. What I expected to happen: [describe it] What actually happens: [describe it] [Paste your current HTML code here] Please find and fix the problem, then give me the full corrected file.
Tip: Describing the gap between "expected" and "actual" is the single most useful thing you can tell a debugging AI.
5
The Explain This Code Prompt Use when: you're curious or want to learn
Explain this HTML file to me in plain language. I have no coding background. Focus on: [the part you're curious about, e.g. "how the timer works"] [Paste your HTML code here]
Tip: Understanding even a little bit of what the code does helps you write better prompts next time. You don't need to understand everything.
🤝
Pair practice (5 min): Have learners pair up and each try Prompt #1 right now with their own scenario idea. They don't need to finish — just get the AI to produce something they can open in a browser.
35 – 65 min

Hands-On: Build Your Own Simulation Tool

30 min

Learners build a simple HTML simulation tool for a clinical scenario relevant to their own program. They'll use Prompt Template #1 to start, then iterate.

📋
Instructions to post or project:
  • 1
    Pick your scenario (3 min)

    Choose a clinical scenario from your own simulation program — or pick one from the idea list below. It should be something you'd actually use.

  • 2
    Write your first prompt (5 min)

    Use Template #1. Fill in the blanks. Be specific about what you want the tool to do and who will use it.

  • 3
    Get the code — save it as a .html file (5 min)

    Copy everything the AI gives you. Open a plain text editor (Notepad on Windows, TextEdit on Mac — set to plain text). Paste it. Save as my-tool.html.

  • 4
    Open it in your browser (2 min)

    Double-click the file, or drag it into a browser window. It should run. If it doesn't, use Template #4 to debug.

  • 5
    Iterate (15 min)

    Use Templates #2 or #3 to improve your tool. Add a feature, change the design, fix something that's off. Aim for at least 2 rounds of iteration.

Reference

Troubleshooting Guide

❌ The file won't open / browser shows an error
Make sure you saved the file with a .html extension (not .html.txt). On Mac TextEdit: Format → Make Plain Text before saving. On Windows: in the Save dialog, set "Save as type" to "All Files" and manually type myfile.html.
❌ The AI gave me only a code block, not a full file
Ask: "Can you give me the complete, self-contained HTML file starting with <!DOCTYPE html> so I can save it directly?" The AI will provide the full thing.
❌ Voice prompts aren't working
Check that your browser isn't blocking audio (look for a 🔇 icon in the address bar). Try Chrome or Edge — they have the most complete Web Speech API support. Safari works on Mac but may need a user gesture first (click anything on the page).
❌ The tool looks broken or the layout is wrong
Use Template #2: describe exactly what looks wrong visually (e.g., "the buttons are overlapping" or "the text is too small on mobile"). Be as specific as you can, then paste your code and ask for the full updated file.
❌ I changed something and now nothing works
Don't edit the code manually. Always work through the AI. If you made changes directly and broke something, paste your current (broken) code and use Template #4 to debug. Alternatively, ask: "Revert this to a working version that does [X]."
❌ The AI keeps stopping halfway through the code
The AI's response hit its limit. Type "Continue" or "Please complete the rest of the file" and it will pick up where it left off. Then combine both halves into one file.

Paste this into any AI chatbot to build your first simulation tool after the workshop:

Create a single-file HTML simulation tool for medical training. Scenario: [describe your scenario in 1–2 sentences] The tool should help: [who will use it and what they need to do] Include: - [feature 1] - [feature 2] - [feature 3] Requirements: works offline, opens in any browser, no external libraries, clean and professional design, mobile-friendly.

Key Principles to Remember

  • You are the expert. The AI knows how to code. You know simulation. That combination is powerful.
  • Iterate, don't perfect. Get something working in 10 minutes, then improve it. Don't wait for a perfect first prompt.
  • One change at a time produces better, more predictable results than big change lists.
  • Save working versions. Before asking for big changes, save a copy of your current file so you can go back.
  • Share what you build. These tools are sharable files. A simulation tool you build today could help a colleague next week.
